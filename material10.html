FASE 1 — El Propósito del Dominio Web (5 pts)

Teoría — Propósito de los Dominios: Los servidores de Internet se localizan mediante cadenas complejas de números binarios o decimales. El propósito fundamental de un Dominio Web es enmascarar esa complejidad para que los seres humanos puedan recordar las direcciones fácilmente.

Escribe el término técnico exacto que define a esa etiqueta numérica única que identifica a un servidor en Internet y que el dominio se encarga de traducir:

$ dominios_sys:~

FASE 2 — Jerarquía Estructural: El Primer Nivel (5 pts)

Teoría — Niveles del Dominio: Los dominios web no son cadenas planas; se leen estrictamente de derecha a izquierda y poseen una jerarquía regulada por la ICANN. La última parte a la derecha representa el nivel organizativo o sectorial más alto del sistema de nombres.

¿Qué siglas en inglés (Top Level Domain) definen a esta parte final derecha de un dominio web que categoriza el propósito global de la página?:

$ dominios_sys:~

FASE 3 — Tipologías ccTLD Territoriales (5 pts)

Teoría — Dominios de Nivel Superior Geográfico: Las extensiones finales se dividen en Genéricas (gTLD como .com) y de Código de País (ccTLD). Estas últimas están compuestas exclusivamente por dos letras y delimitan la soberanía o presencia digital de un territorio geográfico específico.

¿A qué país soberano de Centroamérica le corresponde por estricta norma internacional la extensión territorial exclusiva de dos letras ".pa"?:

$ dominios_sys:~

FASE 4 — El Traductor Universal: Servidores DNS (5 pts)

Teoría — Funcionamiento del DNS: Para que las computadoras entiendan los nombres alfanuméricos, existe una inmensa base de datos distribuida globalmente en miles de servidores que resuelven las peticiones traduciendo de forma transparente los nombres de dominio a direcciones binarias/IP.

¿Cuáles son las siglas en inglés del sistema (Domain Name System) que se encarga de hacer esta traducción automática de nombres a direcciones IP?:

$ dominios_sys:~

CORTAFUEGOS: Verificación de Código Criptográfico

El sistema central requiere validar la palabra clave generada en las primeras 4 fases para poder desplegar el siguiente bloque avanzado de la arquitectura web.

$ security_key:~

FASE 5 — El Punto Raíz o Delimitador (5 pts)

Teoría — Los Delimitadores de Nivel: En la jerarquía e infraestructura técnica real del Sistema de Nombres de Dominio, cada nivel organizativo o nodo de la cadena jerárquica se separa formal y obligatoriamente por un signo gráfico delimitador estandarizado.

¿Qué signo gráfico/ortográfico se utiliza para separar las secciones o niveles de un dominio web (ej: entre el nombre y su extensión)?:

$ dominios_sys:~

FASE 6 — Dominios Comerciales Globales (5 pts)

Teoría — Dominios Genéricos de Primer Nivel (gTLD): Los gTLD agrupan los sitios web según su propósito general sin limitations geográficas. El dominio de este tipo más extendido del planeta nació originalmente orientado a corporaciones y actividades lucrativas de comercio.

¿Qué extensión de tres letras (gTLD) es la más utilizada en el mundo entero para identificar sitios web comerciales y de negocios?:

$ dominios_sys:~

FASE 7 — Organizaciones sin Fines de Lucro (5 pts)

Teoría — Identidad de Organizaciones: Para preservar la naturaleza de las entidades no gubernamentales, fundaciones benéficas, agrupaciones de la sociedad civil y proyectos comunitarios de código abierto, se designó una extensión genérica de tres letras dedicada.

¿Qué extensión de tres letras identifica típicamente a Organizaciones, Comunidades o Fundaciones sin fines de lucro?:

$ dominios_sys:~

FASE 8 — Comercialización y Registro Legal (5 pts)

Teoría — Registradores de Dominios: Los nombres de dominio no se compran en cualquier tienda informática. La ICANN otorga licencias a entidades comerciales específicas para gestionar, dar de alta en la base de datos central y vender la reserva de nombres de dominio al público general.

¿Quién vende un nombre de dominio y se encarga de tramitar su alta comercial oficial bajo supervisión internacional?:

$ dominios_sys:~

Resumen Técnico de Infraestructura

1. Dominios y Enrutamiento: Las direcciones alfanuméricas facilitan la interacción humana enmascarando las IP binarias.


2. Arquitectura DNS: Opera mediante una red distribuida mundialmente encargada de mapear peticiones cliente-servidor.
