Comunicación Windows y Linux mediante Samba

Materia: Informática | Prof. Arquimedes Mosquera

Objetivo

Comprender cómo se realiza la comunicación entre sistemas operativos Windows y Linux utilizando una red local y el servicio Samba.

Escenario de Red:
• Servidor: Linux (Raspberry Pi 3B) | Nombre: server | IP: 192.168.10.110 (Samba e intercambio)
• Clientes: Windows 11 Home estudiantes | PCs (24) | IP: 192.168.10.x

PASO 1 — Identifica tu red (5 pts)

Determina en qué red estás trabajando. Antes de buscar otros equipos es necesario conocer la configuración IP de tu computadora.

  1. Abre la consola CMD de Windows.
  2. Ejecuta el siguiente comando:
C:\> ipconfig

PASO 2 — Descubre equipos activos (5 pts)

Una vez identificada la red, busca dispositivos que respondan dentro de ella.

C:\> for /L %i in (1,1,254) do @ping -n 1 -w 100 192.168.10.%i | find "TTL="
Explicación del comando: Realiza un escaneo básico enviando un único ping a todas las direcciones de la 1 a la 254. El filtro find "TTL=" muestra únicamente los equipos activos cuyas respuestas contienen el tiempo de vida (TTL).

PASO 3 — Identifica posibles servidores Linux (5 pts)

Analiza los valores de TTL obtenidos en el paso anterior para identificar qué equipo corre bajo Linux (el Servidor).

Sistema OperativoTTL AproximadoEstado
Windows128❌ No es el servidor
Linux / Ubuntu64✅ Sí es el servidor
Router64 o 255(Posible puerta de enlace)

PASO 4 — Verificación de Comunicación Específica (5 pts)

Antes de acceder a la carpeta compartida, comprueba la comunicación directa y estable con el servidor IP que descubriste.

C:\> ping 192.168.10.X

PASO 5 — Recursos Compartidos y CasaOS (10 pts)

Conceptos Clave de la Práctica

Samba & SMB Protocolo de red que permite interactuar con sistemas de archivos y carpetas compartidas de Windows desde Linux en total transparencia.
Métrica TTL (Time to Live) Indica los saltos máximos de red. Por defecto, respuestas con TTL cercano a 64 provienen de sistemas basados en Linux, y un TTL de 128 proviene de plataformas nativas Windows.
