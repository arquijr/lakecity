FASE 1 — Fundamentos del Acceso Remoto (5 pts)

Teoría — Objetivo del laboratorio: El acceso remoto permite establecer una conexión desde un equipo Windows hacia un servidor equipo remoto, acceder a su escritorio gráfico, ejecutar comandos y administrar carpetas, sin estar físicamente frente al servidor. Es una de las habilidades más utilizadas en la administración de sistemas: permite a un técnico o administrador de redes gestionar servidores a distancia, y es la base para comprender otras herramientas de acceso remoto como SSH, VNC o TeamViewer.

Escribe la sigla en inglés (tres letras) del protocolo que permite controlar remotamente el escritorio gráfico completo de otro equipo, tal como si estuviera frente a él:

$ acceso_remoto:~

FASE 2 — Parte I: Comando de Conexión en Windows (5 pts)

Teoría — Abrir el cliente de Escritorio Remoto: Se presiona Windows + R para abrir la ventana "Ejecutar", que permite lanzar programas escribiendo su nombre directamente. En el campo de texto se escribe el comando correspondiente y se presiona Enter; esto abre la aplicación oficial de Conexión a Escritorio Remoto de Windows, donde luego se ingresa la IP del servidor (192.168.10.104), el usuario y la contraseña.

¿Qué comando se escribe en la ventana Ejecutar para abrir el cliente de Escritorio Remoto de Windows?:

$ acceso_remoto:~

FASE 3 — Datos de Conexión al Servidor (5 pts)

Teoría — Datos de conexión: Antes de iniciar la conexión se debe contar con la dirección IP del servidor, el sistema operativo remoto, el protocolo utilizado y las credenciales de acceso. En la ventana de Conexión a Escritorio Remoto, en el campo "Equipo", se escribe la dirección IP donde está escuchando el servicio RDP del servidor Debian: ip del equipo remoto, usuario cep y contraseña code2026.

¿Cuál es la dirección IP exacta del equipo remoto que debe escribirse en el campo "Equipo" de la ventana mstsc?:

$ acceso_remoto:~

FASE 4 — El Puerto del Protocolo RDP (5 pts)

Teoría — Protocolo y puerto de RDP: RDP (Remote Desktop Protocol) es el protocolo que utiliza Windows para transmitir la interfaz gráfica y recibir las entradas de teclado/mouse del cliente hacia el servidor remoto. Este protocolo tiene reservado por defecto un número de puerto TCP estándar, el mismo que debe estar abierto en el firewall del servidor Debian 13 para permitir la conexión entrante.

¿Qué puerto TCP utiliza por defecto el protocolo RDP?:

$ acceso_remoto:~

FASE 5 — Identificación del Sistema (5 pts)

Teoría — Información del sistema: Ya dentro del escritorio remoto de Debian 13, se abre una terminal Linux para identificar el sistema operativo y las características de hardware del servidor. Un comando muestra en pantalla, de forma gráfica y resumida, el logotipo de la distribución junto con datos como el sistema operativo, el kernel, la memoria RAM y el shell utilizado.

¿Qué comando de Linux muestra el logotipo de la distribución junto con el resumen del sistema operativo, kernel, RAM y shell?:

$ acceso_remoto:~

FASE 6 — Ampliación 12°: Identidad del Usuario (5 pts)

Teoría — Actualización para 12° grado: Además de la conexión RDP, el estudiante de 12° debe ejecutar comandos adicionales de administración por terminal: hostname, whoami, pwd, ip addr, ls -la, df -h y free -h. Cada uno entrega información distinta del servidor: identidad de equipo, usuario activo, ruta de trabajo, configuración de red, contenido de carpetas, uso de disco y memoria.

¿Qué comando de Linux muestra el nombre del usuario que inició sesión actualmente en la terminal?:

$ acceso_remoto:~

Teoría — Carpeta personal del estudiante: En el equipo remoto existe un directorio compartido llamado Estudiantes, organizado en subcarpetas por grado y sección (12A y 12B). El estudiante debe ingresar a la subcarpeta que corresponde a su grado y sección, y dentro de ella crear una nueva carpeta identificada con su nombre, sin apodos ni abreviaturas.

¿Cuál es el nombre del directorio compartido en el que debes ubicarte (dentro de tu subcarpeta 12A o 12B) antes de crear tu carpeta personal?:

$ acceso_remoto:~

FASE 8 — Seguridad y Administración Remota (5 pts)

Teoría — RDP vs SSH: RDP permite acceso remoto gráfico (se ve el escritorio completo), mientras que existe otro protocolo, ampliamente usado en la administración de servidores Linux, que brinda acceso remoto seguro y cifrado únicamente por línea de comandos (sin interfaz gráfica), siendo más liviano y muy utilizado por administradores de redes para tareas de configuración y seguridad.

¿Qué protocolo de tres letras, a diferencia de RDP, ofrece acceso remoto seguro por línea de comandos y es el estándar en la administración de servidores Linux?:

$ acceso_remoto:~

Resumen Técnico del Laboratorio

1. Acceso Remoto: RDP permite controlar el escritorio gráfico de un servidor Debian 13 desde un cliente Windows (mstsc), usando el puerto TCP 3389.


2. Administración por Terminal: Comandos como whoami, pwd, hostname, ip addr, ls -la, df -h y free -h permiten identificar usuario, ruta, red, archivos, disco y memoria del servidor.


3. Seguridad: A diferencia de RDP (gráfico), SSH ofrece acceso remoto cifrado por línea de comandos, siendo el estándar de administración segura en servidores Linux.
